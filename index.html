HTML

  • Start off with a sturdy frame!


    • HTML stands for "Hypertext Markup Language"

  • It is one of three cornerstone languages used in web development:


    1. HTML
    2. CSS
    3. JS

  • Element: a type of content on a webpage


    • Includes (but not limited to):

      • Text Elements
      • Image Elements
      • Header Elements
      • List Elements

    • Additionally, there are elements used to divide and describe the content on a page including:

      • Meta-Related Elements
      • Content Division Elements
      • Semantic Elements

    • All the elements work together in order to build the structure of the web page and provide framework to make the user experience easier.

    • Elements are typically composed of an opening and closing tag using angle brackets:

        • "< >"


  • <head> element: contains information (metadata) that helps the browser render the page correctly. These elements are not visible to the user when they interact with a web page.


    • Inside the <head> element, we can use the following metadata tags:

      • <title>: Defines the title for a webpage. Required Element. Important for Search Engine Optimization (SEO).
      • <style>: Defines the CSS styles associated with a website.
      • <base>: Defines the base URL for a webpage.
      • <link>: Connects an external resource to the HTML document.
      • <meta>: Defines metadata such as the character set, description, keywords, author, and viewport.

        • Note: List of types of info that can be stored in the <meta> tag


      • <script>: Embeds client-side scripts in the HTML document.

  • <body> element: contains the content that is shown to the user. Any elements that we would like the user to be able to see and interact with must be placed inside this element.


    • Note: Elements that are nested inside other elements are called child elements. The element hosting the nested elements is called the parent element.

    • Also: The <body> element is what is known as a semantic HTML element.

        • Semantic HTML Element: The name of the element describes its content and that role that it plays on a webpage


    • When working with visible HTML elements, the content is always contained between an opening and a closing tag.

      • Usually with < >
      • A closing tag begins with a forward slash:

          • Note: </head>


      Example: <h1>This is some cool text I'll use as a heading!</h1>


    • Examples of child elements within a parent <body> element:

      • <h1> - <h2>: Headings with each number associated to the level of heading
      • <p>: paragraph or block of text
      • <ul>: unordered lists
      • <ol>: ordered lists
      • <li>: list items
      • <img>: contains information about images that are displayed on the webpage (can have several attributes)

          • Note: <src> defines the location of the image


      • <br>: contains a line of empty space
      • <a>: anchor, creates links to the same webpage or other webpages

          • Note: href points to the URL for the link


      • <section>: Defines a section in a document.
      • <id>: used to identify elements and select them.

  • Concluding thoughts on HTML:


    • HTML elements are case sensitive
    • Declarations are not case sensitive.

        • For example:

          <!DOCTYPE html>


    • HTML attributes are always defined in the beginning element, never the closing element.

        • For example:

          You see <html lang=”en”> at the beginning of the HTML document and you see that a language attribute of “English” was designated at the very beginning!


    • HTML helps us build a "sturdy frame" right off the bat!

Git

  • Manage and keep track of a project!


    • Access via command line

    • Commands always start with "git"

      • git status: checks what branch we are currently on
      • git checkout -b branch-name: creates a new branch and switches to it

        • Refer to: Git Commands Cheat Sheet for more commands!


    • Used to save and track versions of code

    • Initiate (create) a repository, add code to repository, view different versions of our code.

    • Good for tracking changes locally.

    • To collaborate, use GitHub:

      • Stored remotely for collaboration
      • Cloud based technology
      • Other tools that can be used include Bitbucket and SourceForge which are Git-based repository hosting.
